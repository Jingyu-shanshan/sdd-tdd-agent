# Acceptance criteria

- [x] JSON commands exchange the exact typed design request and proposal.
- [x] Response keys and types are strictly validated.
- [x] Codex runs ephemerally with read-only sandbox and strict output Schema.
- [x] Codex uses the injected executable resolver and private temporary files.
- [x] Errors reveal no request, model output, stderr, credentials, or temp paths.
- [x] Adapters never modify Session or project files.
- [x] Existing requirement adapters preserve their behavior.
- [x] No dependency or `.gitignore` update is required.
- [x] Active product Session remains unchanged.
