# Requirement: Show project and session status

## Description

Add `agent status` so a developer can observe initialized project metadata and
the active SDD/TDD session without opening internal files manually.

## User story

As a developer using the platform, I want one status command so I can confirm
the detected toolchain and see which workflow state is active.

## Functional requirements

- Read scalar project metadata from `.agent/project.yml`.
- Read the `test_frameworks` list written by `agent init`.
- Read `current_session` when present.
- For an active session, read its `state` from
  `.agent/sessions/<session-id>/state.json`.
- Reject session identifiers containing path separators or dot-directory
  traversal before reading a state file.
- `agent status` writes these lines in this exact order:

  ```text
  Project: <name>
  Language: <language-or-unknown>
  Build tool: <tool-or-unknown>
  Test frameworks: <comma-separated-or-none>
  Session: <session-id-or-none>
  State: <state-or-none>
  ```

- Successful status output exits with code 0.

## Non-functional requirements

- Use only the standard library at runtime.
- Return an immutable typed status object from the reading layer.
- Keep file reading independent from CLI output formatting.
- Support the YAML subset generated by this platform; general YAML parsing is
  outside this increment.
- All `AGENTS.md` quality gates must pass.

## Deferred scope

- Status before `.agent/project.yml` exists.
- Invalid YAML or JSON and missing active-session state files.
- Colored or interactive terminal output.
- Session progress percentages and metrics.
