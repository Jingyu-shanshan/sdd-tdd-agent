# Requirement

## Goal

Close the implemented SDD and incremental TDD roadmap with one honest,
reproducible release-readiness audit.

## Requirements

- Reconcile completed task evidence with observed hosted CI results.
- Remove stale architecture statements that contradict implemented behavior.
- Validate the complete repository through formatting, linting, typing, tests,
  packaging, state, security, and ignore-policy gates.
- Preserve the user's active `support-pdf-export` Session byte-for-byte.
- Keep GitHub Copilot planned until its JSONL event fields have a documented,
  bounded, contract-tested adapter.
- Keep Linux Mint real-host validation pending until an explicitly labelled
  self-hosted runner produces evidence.

## Non-goals

- Adding a speculative Provider adapter or unsupported ecosystem.
- Claiming external platform evidence that has not been observed.
- Changing production behavior or existing tests during documentation closeout.
